# 协作网络：P2P 模型

> **2026-05-02**：协作网络从中心化的 Controller → Runtime 调度模型，改为去中心化的 P2P 直连模型。

## P2P 网络

每个 Pudding Agent 启动后自动加入 P2P 网络：

1. **节点发现**：通过 mDNS 或 UDP 广播发现局域网内的其他 Agent
2. **直连通信**：Agent 之间建立 HTTP/gRPC 连接
3. **事件广播**：Agent 将自身状态变化和任务完成事件广播给感兴趣的节点
4. **任务协作**：Agent 可以请求其他 Agent 执行子任务

## 数字蚁群模型

- 每个 Agent 像蚂蚁一样感知局部状态
- 通过事件痕迹（事件广播）协作
- 无需中心调度器
- 任务图支持动态发现、拆分、合并

## Workspace

Workspace 仍然是治理边界：
- 每个 Agent 有本地 Workspace 配置
- 同 Workspace 的 Agent 共享事件域
- Workspace 配置可同步
