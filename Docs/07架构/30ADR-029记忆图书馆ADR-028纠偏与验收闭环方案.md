# 30 ADR-029 记忆图书馆 ADR-028 纠偏与验收闭环方案

> 状态：**proposed**
> 日期：2026-05-21
> 范围：ADR-028 终验纠偏、MemoryLibrary FTS、workspace scope、schema 初始化、Pointer 泛化、MemoryLibrarian 分层、Recall source-aware 输出、验收测试
> 前置：[29ADR-028记忆图书馆基础设施重构ADR](29ADR-028记忆图书馆基础设施重构ADR.md)

---

## 1. 背景

ADR-028 已被标记为 `implemented`，但终验发现该状态不成立：

1. `PuddingMemoryEngineTests` 当前失败，基础 FTS 用例不通过。
2. Runtime 工具链虽然在 `SkillInvokeRequest` 中携带 `WorkspaceId`，但 memory tools 没有消费该字段，正常工具调用仍会落到 `default`。
3. `grep_memory` 在 scoped FTS 无命中时会回退到 unscoped `SmartSearchAsync`，存在跨 workspace 召回风险。
4. EF Core model 已增加 `SourceReferences`、`MemoryTreeNodes`、`BookTreeMounts`，但 `init_library.sql` 未同步，已有数据库和 SQL 初始化路径会漂移。
5. Pointer 仍是 chapter-only，没有完成 ADR-028-F 的 generalized pointer。
6. `MemoryLibrarian` 仍委托 `MemoryLibraryConvenience`，`SubconsciousOrchestrator` 仍直接写 convenience，分层只是名义落地。
7. `RecalledMemory.Sources` 已存在，但 `MemoryRecallService` 没有填充来源摘要。

本 ADR 的目标不是扩大 ADR-028，而是把 ADR-028 收敛到可验证、可回归、可接受的完成状态。

---

## 2. 纠偏决策

### ADR-029-A：ADR-028 状态必须回滚为 partial，直到验收通过

**决定**：将 ADR-028 文档和上下文中的状态从 `implemented` 改为 `partially-implemented`。

ADR-028 只有在下列条件全部满足后才可重新标记为 `implemented`：

- `PuddingMemoryEngineTests` 通过。
- 新增 workspace scope 回归测试通过。
- 新增 schema 初始化回归测试通过。
- Pointer 新旧 API 兼容测试通过。
- Subconscious 写入路径经过 `IMemoryLibrarian`。
- Recall 输出包含 source summaries。

原因：

- 文档状态必须服务工程判断，不能把“提交了代码”当作“完成验收”。
- 后续 Dev / Agent 需要基于真实状态继续工作。

### ADR-029-B：workspaceId 由 Runtime 注入，LLM 参数只作为兼容输入

**决定**：memory tools 的有效 workspace 选择规则固定为：

1. 优先使用 `SkillInvokeRequest.WorkspaceId`。
2. 当 `SkillInvokeRequest.WorkspaceId` 为空时，才读取参数中的 `workspace_id`。
3. 两者都为空时才 fallback 到 `default`，并记录 warning。

工具 schema 可以继续保留 `workspace_id` 作为兼容参数，但系统不能依赖 LLM 正确传入它。

原因：

- workspace scope 是运行时安全边界，不是 prompt 层约定。
- 用户和 agent 不应该通过自然语言参数决定数据隔离。

### ADR-029-C：运行时检索不得从 scoped 路径回退到 unscoped 路径

**决定**：`grep_memory`、`MemoryRecallService`、Context L6 等运行时路径禁止调用 unscoped `SmartSearchAsync` 作为 fallback。

允许：

- 新增 `SmartSearchScopedAsync(workspaceId, query, topK)`。
- 或者只使用 `SearchChaptersFtsScopedAsync` / `ListBooksScopedAsync` 等 scoped primitive。
- 管理员跨 workspace 检索必须使用显式 API 和显式参数。

禁止：

- 当前 workspace 无命中时自动搜索所有 workspace。
- 在运行时路径里静默调用旧 unscoped API。

### ADR-029-D：schema 初始化必须与 EF model 同步

**决定**：`init_library.sql` 必须补齐 ADR-028 新结构，并保持幂等：

- `Chapters.SourceReference`
- `Chapters.ReferenceType`
- `SourceReferences`
- `MemoryTreeNodes`
- `BookTreeMounts`
- generalized `Pointers` 所需新增列

SQLite 初始化和已有数据库升级都必须可重复执行。没有 EF migrations 的前提下，本项目继续使用幂等 `CREATE TABLE IF NOT EXISTS` 与受控 `ALTER TABLE`。

原因：

- 当前 `EnsureCreatedAsync()` 不能迁移已有数据库。
- SQL 初始化文件是生产和测试外的另一条真实初始化路径。

### ADR-029-E：Pointer 先双写兼容，再切换语义

**决定**：Pointer 泛化采用兼容迁移：

目标字段：

```text
Pointers
  PointerId
  WorkspaceId
  SourceType
  SourceId
  ChapterId         legacy compatibility
  TargetType
  TargetId
  TargetLabel
  Description
  Relevance
  CreatedAt
```

兼容规则：

- 旧 `CreatePointerAsync(chapterId, ...)` 保留，并写入 `SourceType = "chapter"`、`SourceId = chapterId`、`ChapterId = chapterId`。
- 新增 `CreatePointerAsync(workspaceId, sourceType, sourceId, targetType, targetId, ...)`。
- 旧 `GetPointersAsync(chapterId)` 映射到 `SourceType = "chapter" && SourceId = chapterId`，兼容历史 `ChapterId`。
- 新增 `GetPointersAsync(workspaceId, sourceType, sourceId)`。

原因：

- Pointer 是基础设施接口，不能一次性破坏现有测试和调用方。
- 双写能让旧数据和新数据在同一查询模型下被召回。

### ADR-029-F：MemoryLibrarian 必须成为潜意识写入入口

**决定**：潜意识写入图书馆的链路必须变为：

```text
SubconsciousOrchestrator
  -> IMemoryLibrarian
    -> IMemoryLibrary
```

`MemoryLibraryConvenience` 保留为兼容 facade，但不能作为潜意识主写入路径。

要求：

- `MemoryLibrarian.IngestExperienceAsync` 直接调用 `IMemoryLibrary` primitive。
- 写入 chapter 后同步创建 `SourceReference`。
- 需要目录挂载时调用 `EnsureDefaultBooksAsync` / `CreateTreeNodeAsync` / `MountBookAsync`。
- 未实现的 tree operation 类型必须返回明确结果或抛出受控异常，不得只记录 `not yet implemented` 后继续宣称完成。

### ADR-029-G：Recall 输出必须带 source summaries

**决定**：`MemoryRecallService` 对 library result 映射为 `RecalledMemory` 时，必须读取 chapter 对应的 `SourceReferences` 并填充 `Sources`。

要求：

- 对每个召回 chapter 调用批量或局部 source lookup。
- `SourceReference` / `ReferenceType` 的 inline 字段可作为兼容 fallback。
- source 解析失败不应阻断召回，但必须保留 `TargetType`、`TargetId` 和 `ResolveStatus`。

原因：

- ADR-028 的核心目标是“提纯记忆可回溯到原始会话/资源”。
- 召回不携带 source，显意识 LLM 无法按需还原上下文。

- 召回排序不因 source lookup 改变。

验收：

- 测试证明召回结果可回溯到 `session` / `session_event` / `url` 等来源。

## 4. 完成定义

```powershell
dotnet build Source\PuddingAgent\PuddingAgent.csproj --no-restore --nologo
```

- `MemoryToolsWorkspaceTests`
- `MemoryLibrarySchemaTests`
- `MemoryLibrarianTests`
- `MemoryRecallServiceTests`
- generalized pointer tests

## 5. 非目标

本 ADR 不做：

- 删除 `MemoryFacts` / `MemoryPreferences`。
- 设计新的 LLM 提示词整理策略。
- 引入 EF migrations。
- 做跨 workspace 管理员搜索 UI。
- 改造所有历史数据到 TreeNode。

这些内容需要后续单独 ADR。
