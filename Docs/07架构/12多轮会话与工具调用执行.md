# 多轮会话与工具调用

> **2026-05-02**：执行模型不变——Runtime 驱动 loop，LLM 只做单轮决策。但 Runtime 是同进程内模块。
> **2026-05-03**：Chat 流式响应新增 SSE 全栈穿透路径，每层独立 relay，支持实时打字机效果与可中断生成。

## SSE 流式路径（Chat）

```
Platform ChatApiController (SSE endpoint /api/workspaces/{wsId}/chat/message/stream)
  → PlatformApiClient.ReadSseFramesAsync
    → Controller MessageIngressController (stream)
      → SessionRouter.RouteMessageStreamAsync
        → RuntimeDispatcher.DispatchStreamAsync
          → RuntimeExecuteController.ExecuteStream
            → AgentExecutionService.ExecuteStreamAsync
              → IRuntimeLlmClient.ChatStreamAsync
                → LlmProxyController.ChatStream
                  → OpenAiLlmGateway.ChatStreamAsync (OpenAI SSE)
```

关键设计：
- 每层定义独立 SSE 端点，上游消费 `ServerSentEventFrame` 后 relay 给上一级
- 每层独立管理 `CancellationToken`，用户点击停止 → `AbortController` → 逐级 `OperationCanceledException`
- 帧类型通过 SSE `event:` 字段区分：`delta`（增量文本）、`usage`（Token 统计）、`error`、`done`
- Token 用量以 `TokenUsageDto` 全链路透传，不做中途变换
- 公共 SSE 解析逻辑提取到 `StreamingContracts.SseFrameReader`

## 执行模型（同步路径）

- Runtime 模块控制循环
- LLM 每次只做单轮决策
- 工具在进程内直接执行
- 停止信号由 Controller 模块（同进程）下发

## 与之前设计的唯一区别

同步路径中不再有 Controller → Runtime 的 HTTP 调用，改为同进程内方法调用，延迟更低。流式路径因需要逐层 relay SSE 帧，仍走 HTTP 通道。
