# 事件系统 V2 — 统一事件总线、优先级队列与中断

> **ADR-016** | **2026-05-12** | Lead 发起设计

## 背景

V1 事件系统仅覆盖两个窄场景：
- `IStreamingEventBus`：SSE 推前端（thinking/delta/tool_call）
- `ICoordinationEventBus`：P2P 节点间锁/发现广播

缺少一个**统一内部事件总线**来解耦以下关注点：
- Cron 定时触发 → 任务执行 → 结果回写
- 外部 Connector（MQTT/Webhook/Email）入站事件 → 优先级路由 → Agent 处理
- Agent 生命周期（睡眠/唤醒/心跳）
- 多源事件预处理（去重/批处理）
- 事件订阅（Agent 主动 subscribe/unsubscribe）

## 核心抽象

### 1. 事件模型 `InternalEvent`

```
InternalEvent {
    EventId: string (GUID)
    Type: string               // "cron.trigger", "mqtt.sensor", "agent.subscribe" ...
    Priority: enum { Normal=0, Important=5, Urgent=10 }
    Isolation: enum { Mainline, Isolated }
    Source: EventSource        // { SourceType, SourceId, ConnectorId }
    SessionId: string?         // 主线模式有值
    Payload: object (JSON)
    Timestamp: long (Unix ms)
    Metadata: dict<string,string>
}
```

### 2. 事件总线 `IInternalEventBus`

进程内 pub/sub，支持类型过滤：
```
IInternalEventBus {
    PublishAsync(InternalEvent) → Task
    SubscribeAsync(string eventType, Func<InternalEvent, Task>) → ISubscription
    UnsubscribeAsync(ISubscription) → Task
}
```

### 3. 优先级队列 `IPriorityEventQueue`

```
IPriorityEventQueue {
    EnqueueAsync(ProcessedEvent, PriorityLevel) → Task<string>
    DequeueAsync(ct) → Task<QueuedEvent?>
    PeekAsync(count, ct) → Task<IReadOnlyList<QueuedEvent>>
    GetStatsAsync(ct) → Task<QueueStats>
}
```

三级队列语义：
| 级别 | 值 | 行为 | 场景 |
|------|---|------|------|
| Normal | 0 | FIFO 顺序，不打断 | 日报生成、数据同步 |
| Important | 5 | 插队，会话间隙注入，保留现场 | 门禁请求、重要通知 |
| Urgent | 10 | 立即打断，最快路径 | 火警/水浸/摔倒 |

### 4. 事件预处理 `IEventPreprocessor`

```
IEventPreprocessor {
    ProcessAsync(RawEvent[]) → Task<ProcessedEvent[]>
}
策略（按 Connector 可配置）：
- DedupWindow：相同 sourceId+eventType → 保留最后一个
- BatchWindow：相同 sourceId 不同 eventType → 打包
```

### 5. Agent 现场保存/恢复 `IAgentCheckpointService`

```
IAgentCheckpointService {
    SaveCheckpointAsync(SessionId, CallStack, PendingTools) → Task<Checkpoint>
    RestoreCheckpointAsync(SessionId) → Task<Checkpoint?>
    DeleteCheckpointAsync(SessionId) → Task
}
```

### 6. 事件订阅工具 `IEventSubscriptionTool`

Agent 通过工具调用 subscribe/unsubscribe：
```
事件订阅工具 {
    subscribe_events(eventTypes: string[], filter?: string)
    unsubscribe_events(subscriptionId: string)
    list_subscriptions()
}
```

## 数据流全景

```
外部世界 (IoT/Webhook/Email/MQTT)
    │
    ▼
IPuddingConnector.OnEventReceived()
    │
    ▼
IEventPreprocessor (去重+打包)
    │
    ▼
IPriorityEventQueue (持久化+优先级入队)
    │
    ▼
EventDispatcher (优先出队 + 路由)
    │
    ├── Mainline  → 追加到当前会话
    └── Isolated  → Fork 临时会话，完成后丢弃
    │
    ▼
AgentExecutionService (ExecuteAsync / InterruptAsync)
    │
    ▼
IInternalEventBus.Publish("event.completed")
    │
    ▼
EventDiagnosticLog (完整生命周期日志)
```

## 打断恢复流程 (Urgent)

```
Agent 正在执行
    │
    ▼ (Urgent Event 到达)
1. SaveCheckpoint(SessionId, CallStack, PendingTools)
2. Inject UrgentEvent → AgentExecutionService
3. Agent 处理紧急事件
4. 事件完成
5. RestoreCheckpoint(SessionId)
6. Agent 从断点继续
```

## 事件订阅工具（Agent 主动能力）

Agent 可通过 `subscribe_events` / `unsubscribe_events` 工具管理事件订阅：

```
subscribe_events:
  - eventTypes: ["mqtt.sensor.*", "cron.daily-summary"]
  - filter: "workspace=default&priority>=5"  (可选)
  → 返回 subscriptionId

unsubscribe_events:
  - subscriptionId: "sub-abc123"

list_subscriptions:
  → 返回当前 Agent 的所有活跃订阅
```

## 持久化存储

新增 SQLite 表（在 `pudding_memory.db`）：

```sql
CREATE TABLE EventQueue (
    Id TEXT PRIMARY KEY,
    Priority INT NOT NULL,
    EventType TEXT NOT NULL,
    SourceType TEXT,
    SourceId TEXT,
    Payload TEXT NOT NULL,
    Status TEXT DEFAULT 'pending',
    CreatedAt INTEGER NOT NULL,
    StartedAt INTEGER,
    CompletedAt INTEGER,
    RetryCount INT DEFAULT 0,
    ErrorMessage TEXT
);

CREATE TABLE EventDiagnosticLog (
    LogId TEXT PRIMARY KEY,
    EventId TEXT NOT NULL,
    Stage TEXT NOT NULL,
    Timestamp INTEGER NOT NULL,
    Detail TEXT,
    DurationMs INTEGER
);

CREATE TABLE AgentCheckpoints (
    CheckpointId TEXT PRIMARY KEY,
    SessionId TEXT NOT NULL,
    AgentId TEXT NOT NULL,
    WorkspaceId TEXT NOT NULL,
    CallStack TEXT NOT NULL,
    PendingTools TEXT,
    ContextSnapshot TEXT,
    CreatedAt INTEGER NOT NULL,
    Status TEXT DEFAULT 'active'
);

CREATE TABLE EventSubscriptions (
    SubscriptionId TEXT PRIMARY KEY,
    AgentId TEXT NOT NULL,
    WorkspaceId TEXT NOT NULL,
    EventTypePattern TEXT NOT NULL,
    FilterExpression TEXT,
    CreatedAt INTEGER NOT NULL,
    Status TEXT DEFAULT 'active'
);
```
