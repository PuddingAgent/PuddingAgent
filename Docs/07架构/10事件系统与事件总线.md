# 事件系统 V3 — 纯管道 · 万物皆事件

> **ADR-016** | **2026-05-12** 初始 | **2026-05-14 V3 重构**

## 核心原则

**事件系统是纯管道。** 它不关心事件来自哪里、去往何处。

```
所有外部系统（Cron、Connector、P2P、System、Message）
    │
    │ IInternalEventBus.PublishAsync(evt)
    ▼
┌──────────────────────────────────────────┐
│            事件系统 （纯管道）              │
│                                          │
│  IEventPreprocessor (去重/打包)           │
│       ↓                                  │
│  IPriorityEventQueue (持久优先级队列)      │
│       ↓                                  │
│  EventDispatcher (BackgroundService)      │
│       ↓                                  │
│  IEventHandler.HandleAsync() ← 唯一边界   │
│                                          │
│  事件系统只知道 IEventHandler，            │
│  不知道 Cron、Agent、Webhook 的存在        │
└──────────────────────────────────────────┘
```

## 背景

V2 事件系统存在结构性问题：
- 数据流图直接引用 `Connector.OnEventReceived()`、`AgentExecutionService`、`External World` 等外部概念
- `EventDispatcher` 注释要求"触发 AgentExecutionService.InterruptAsync"，但事件系统不应知道 Agent 执行引擎
- 文件清单将 `CronSchedulerService`、`AgentExecutionService` 列为"重构/扩展"目标，违反了依赖方向

V3 修正：通过 `IEventHandler` 接口建立清晰边界，事件系统不依赖任何外部模块。

## 核心抽象

### 0. `IEventHandler` — 事件的唯一边界 ⭐ 新增

```csharp
public interface IEventHandler
{
    /// 关注的事件类型通配符，如 "cron.*", "agent.*"
    string EventTypePattern { get; }

    /// 处理事件，返回 true 成功 / false 失败（触发重试）
    Task<bool> HandleAsync(InternalEvent evt, CancellationToken ct);

    /// 是否可被更高优先级事件打断
    bool SupportsInterruption { get; }
}
```

这是事件系统唯一需要知道的「外部」接口。所有消费者实现此接口。

### 1. 事件总线 `IInternalEventBus`

进程内 pub/sub — 事件**发布入口**。任何模块都可以调用 `PublishAsync` 来产生事件。

### 2. 优先级队列 `IPriorityEventQueue`

三级优先级：Urgent(10) > Important(5) > Normal(0)
SQLite 持久化入队，进程重启不丢事件。

### 3. 事件预处理 `IEventPreprocessor`

去重窗口 + 批处理窗口。按 Connector 可配置策略。

### 4. EventDispatcher（BackgroundService）

```csharp
while(true):
    evt = DequeueAsync()     // 优先级出队
    handlers = MatchHandlers(evt.Type)  // 按 EventTypePattern 匹配
    for each handler:
        success = handler.HandleAsync(evt, ct)
    if !success → retry (指数退避, 最多3次) → dead_letter
```

### 5. 事件订阅 `IEventSubscriptionTool`

Agent 通过工具调用 subscribe/unsubscribe 事件类型。显式订阅与隐式订阅均通过此工具管理。

## 数据流全景（V3 修正）

```
┌──────────── 事件来源（任意，事件系统不感知） ────────────┐
│                                                          │
│  CronScheduler  WebhookConnector  P2P  System  Message   │
│       │              │            │     │        │       │
│       └──────────────┴────────────┴─────┴────────┘       │
│                          │                                │
│              IInternalEventBus.PublishAsync(evt)          │
└─────────────────────────┬──────────────────────────────┘
                          │
┌─────────────────────────▼──────────────────────────────┐
│                  事件系统 （纯管道）                       │
│                                                          │
│  EventIngressBridge (IHostedService)                     │
│    · 订阅 IInternalEventBus("*")                        │
│    · RawEvent → IEventPreprocessor.ProcessAsync()        │
│    · ProcessedEvent → IPriorityEventQueue.EnqueueAsync() │
│                          │                                │
│                          ▼                                │
│  IPriorityEventQueue (SQLite 持久化)                      │
│    · Urgent 队列 → Important 队列 → Normal 队列           │
│                          │                                │
│                          ▼                                │
│  EventDispatcher (BackgroundService)                     │
│    while(true):                                          │
│      qe = DequeueAsync()                                │
│      handlers = MatchHandlers(qe.EventType)              │
│      foreach handler: handler.HandleAsync(evt, ct)       │
│      success? complete : retry/dead_letter               │
└─────────────────────────┬──────────────────────────────┘
                          │
                    IEventHandler.HandleAsync()
                          │
┌─────────────────────────▼──────────────────────────────┐
│                事件消费者 （外部）                         │
│                                                          │
│  AgentEventHandler → AgentExecutionService               │
│  (未来) NotificationHandler → 推送通知                   │
│  (未来) AuditLogHandler → 审计日志                       │
│  (未来) CustomHandler → 用户自定义                       │
└──────────────────────────────────────────────────────────┘
```

## 打断恢复流程 (Urgent)

```
Agent 正在执行
    │
    ▼ (Urgent Event 到达)
1. EventDispatcher 判断当前 handler.SupportsInterruption
2. IAgentCheckpointService.SaveCheckpoint()
3. AgentEventHandler.HandleAsync(urgentEvent)
4. 紧急事件处理完成
5. IAgentCheckpointService.RestoreLatestAsync()
6. Agent 从断点继续
```

> 注意：打断逻辑由 EventDispatcher 协调，但真正的 Save/Restore 由 AgentEventHandler 调用 Agent 系统的 CheckpointService 完成。

## 显式订阅 vs 隐式订阅

| 类型 | 描述 | 示例 |
|------|------|------|
| **显式订阅** | Agent 通过 `subscribe_events` 工具主动注册 | `subscribe_events(["mqtt.sensor.temp"])` |
| **隐式订阅** | Agent 异步执行操作时自动注册回调 | 异步命令完成通知、子代理完成通知 |

隐式订阅由 AgentExecutionService 在执行长耗时命令/子代理时自动向 IInternalEventBus 注册临时 handler，事件到达后通过 EventSubscriptionTool 清理。

## 持久化存储
