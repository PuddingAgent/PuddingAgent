# V1 落地与验收

## 第一条垂直切片

第一条必须跑通的真实链路是：

CLI -> Controller API -> Workspace 路由 -> ServiceSession -> Runtime Agent -> 真实 LLM 回复

## 第一阶段优先级

1. 先落地 Controller，建立统一治理入口。
2. 打通渠道消息到 ServiceSession 的路由。
3. 让 Runtime 中指定 Agent 返回真实 LLM 回复。
4. 补上审批、审计、状态查询和调试查询。
5. 再扩展低风险任务、Workflow、知识能力与更复杂治理能力。

## 首批验收标准

- 渠道消息能命中 Workspace 与 AgentTemplate。
- Session 能自动创建或复用。
- Runtime 中 Agent 能返回真实回复。
- 路由决策、Session 状态、审计事件可查询。
- 高风险动作进入审批链时有明确状态与错误输出。

## 当前已知实现差距

- 代码仍保留较多旧的 CLI-first 原型结构。
- Runtime 相关组件尚未完整从 CLI 中迁移出来。
- PuddingController、PuddingAvalonia 等仍以架构设计为主，尚未完整落地。
