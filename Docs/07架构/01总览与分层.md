# 总览与分层

> **2026-05-02 简化**：Controller/Runtime/Platform/Gateway 不再作为独立进程存在。每个 Pudding Agent 是单进程，内嵌全部模块。Agent 之间通过 P2P 协作。

## 产品定位

Pudding Agent Network 不是一套需要部署多个微服务的"平台"。

它是**一个可执行文件**。

> **Pudding = 可 P2P 组网的 AI Agent 进程**

## 单进程内部分层

```
┌────────────────────────────┐
│        Web UI              │  ← 内嵌前端（React，通过 Kestrel 直接服务）
├────────────────────────────┤
│     Controller 模块         │  ← 路由、鉴权、调度、审计（同进程内调用）
├────────────────────────────┤
│     Runtime 模块           │  ← LLM 对话、工具执行、记忆引擎、Skill
├────────────────────────────┤
│     P2P 网络层             │  ← 节点发现、直连通信、事件广播
├────────────────────────────┤
│     SQLite 数据层          │  ← 配置、会话、记忆、审计
└────────────────────────────┘
```

这与原来的分层逻辑完全一致，只是从"进程间 RPC"变成了"进程内方法调用"。

## P2P 协作模型

- 每个 Agent 对等，无中心节点
- 基于 mDNS/UDP 广播自动发现局域网内的对等 Agent
- Agent 之间直接建立 HTTP/gRPC 连接
- 事件在 Agent 之间直接传播（替代 RabbitMQ）
- 数字蚁群模型：每个 Agent 感知局部状态，协作推进任务图

## 部署模型

| 方式 | 说明 |
|---|---|
| 裸进程 | 下载可执行文件，双击运行。Windows/Linux/macOS |
| Docker | `docker run -p 8080:8080 pudding-agent`，单容器 |
| systemd | Linux 服务器后台托管 |

用户不需要安装 PostgreSQL、Redis、RabbitMQ、Nginx、MinIO。
