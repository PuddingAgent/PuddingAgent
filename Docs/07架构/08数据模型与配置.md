# 数据模型与配置

## 核心数据实体

- AgentTemplate：角色、默认能力、运行画像、记忆策略。
- AgentInstance：模板派生出的实际运行实例。
- Session：ServiceSession、ExecutionSession、AuditSession、ChildSession。
- Workflow / WorkflowStep：显式工作流与步骤定义。
- MemoryRecord / MemoryPolicy：正式记忆、候选记忆与记忆治理策略。
- ApprovalRecord：高风险执行与确认链路的核心对象。
- ChannelBinding：渠道绑定到 Workspace 的治理关系。

## 建模原则

- 模板与实例必须分离。
- Workspace 是配置与治理边界，Session 是运行时边界。
- 记忆必须同时建模正式存储与候选提升。
- 路由、审批、权限、审计需要有可查询的显式对象。

## Workspace 最小配置对象

- AgentTemplate
- ChannelBinding
- Workflow
- MemoryPolicy
- PermissionPolicy
- BudgetPolicy
- PluginBinding

## 配置来源

第一阶段采用 Workspace YAML 加平台持久化的混合模式。配置变更优先手动 reload，而不是一开始就做自动热更新。
