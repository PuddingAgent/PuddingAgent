# 数据模型与配置

## 核心数据实体

- AgentTemplate：角色、默认能力、运行画像、记忆策略。
- AgentInstance：模板派生出的实际运行实例。
- SkillRegistryEntry / McpRegistryEntry：全局 Skill / MCP 注册对象。
- Session：ServiceSession、ExecutionSession、AuditSession、ChildSession。
- WorkflowDefinition / WorkflowNode / WorkflowEdge / WorkflowTrigger / WorkflowRun：显式工作流、节点、边、触发器与运行实例定义。
- MemoryRecord / MemoryPolicy：正式记忆、候选记忆与记忆治理策略。
- ApprovalRecord：高风险执行与确认链路的核心对象。
- ChannelBinding：渠道绑定到 Workspace 的治理关系。
- EventEnvelope / EventRoute：统一事件对象与路由结果。
- EventSubscription：Agent、Runtime、审计链路对事件的订阅声明。
- EventCursor / ReplayCheckpoint：长连接消费、补偿消费与重放游标。
- DeadLetterEvent：无法投递、无法消费或超限事件的隔离对象。
- RuntimeNodeDescriptor / RuntimeNodeHeartbeat：Runtime 节点静态画像与动态心跳对象。
- RuntimePlacementPolicy / RuntimePlacementDecision：节点选址规则与选址结果对象。
- SandboxBinding / SandboxInstanceRecord / SandboxFault：沙箱绑定、实例台账与故障对象。
- TaskBoardItem / TaskBoardColumn：共享任务板对象。
- SpreadsheetSheet / SpreadsheetRangeRef：共享表格对象。
- WikiEntry / KnowledgePageVersion：共享知识页面对象。
- ObjectAsset / ObjectAssetVersion：对象存储中的文件与版本对象。
- AgentActivitySnapshot / TaskMapSummary / KeyEventRecord：Workspace 协作驾驶舱的聚合视图对象。
- AgentAccessToken：Agent 通过 HTTP API 访问共享工具时的身份令牌对象。
- AgentExecutionLog / EventLogRecord / WorkspaceLogProjection：Workspace 日志系统的局部日志、事件日志与聚合日志对象。
- LogQuery / LogRetentionPolicy：日志查询条件与日志保留策略对象。

## 建模原则

- 模板与实例必须分离。
- Workspace 是配置与治理边界，Session 是运行时边界。
- 记忆必须同时建模正式存储与候选提升。
- 路由、审批、权限、审计需要有可查询的显式对象。
- 事件模型必须同时支持外部接入事件与内部状态事件，不能只覆盖消息渠道。
- 订阅关系、事件可见性与重放权限必须可治理、可审计、可查询。

## 配置来源

第一阶段采用 Workspace YAML 加平台持久化的混合模式。配置变更优先手动 reload，而不是一开始就做自动热更新。

其中：

- 全局 Skill / MCP Registry 建议由 Controller 管理并持久化到 PostgreSQL。
- Workspace 内的 AgentTemplate 引用这些全局能力，而不是复制它们的定义。

## 事件系统最小配置对象

- `EventPolicy`：声明允许进入本 Workspace 的事件类型、来源范围、信任等级和保留策略。
- `EventSubscriptionPolicy`：声明哪些 Agent / Workflow / Runtime 组件可以订阅哪些事件，以及是否允许直接唤醒。
- `EventRetentionPolicy`：声明事件日志、死信事件、重放窗口和归档策略。
- `AdapterEventMapping`：声明某个 Gateway Adapter 的原始协议字段如何映射到统一事件类型。
