# task25 - Workflow / TaskMap 设计与执行

最后更新：2026-03-18

## 任务目标

把工作流正式提升为 Pudding 的一等执行模型，支持：

- 在平台管理端以可视化画布编辑工作流
- 在后端以 YAML / 结构化定义表达工作流
- 在运行时支持程序节点、组件节点、Agent 节点、事件节点与存储节点混编
- 支持工作流被 Agent 触发、被事件触发，以及工作流触发另一个工作流

## 前端设计参考

前端或管理页面可以参考 FlowGram.AI 的设计理念，重点借鉴：

- 自由布局 / 固定布局画布
- 节点表单配置
- 变量面板与作用域管理
- 结构化节点与边保存

但需要明确：

- FlowGram 更适合做设计态参考
- Pudding 的运行时语义、权限治理、Agent 生命周期和事件驱动执行仍以自身架构为准

## 工作流节点类型

第一阶段建议支持：

1. **程序逻辑节点**
	- 内置组件
	- Python 脚本
	- Shell / Bash
	- Curl / HTTP 请求
	- 其他插件或程序节点

2. **Agent 节点**
	- 长期 Agent
	- 工作流记忆 Agent
	- 一次性 Agent

3. **事件节点**
	- 事件触发器
	- 事件发布器
	- 事件消费者

4. **控制节点**
	- Start
	- End
	- Condition
	- Loop
	- Retry
	- SubWorkflow

5. **存储节点**
	- 知识库写入
	- 表格 / 数据表写入
	- 文件输出

## 工作流触发方式

工作流至少应支持：

- Agent 直接触发
- 事件触发
- 用户 / 平台手动触发
- 工作流触发另一个工作流

## 一个典型流程示例

例如：

`开始 -> 采集 -> 程序或脚本清洗 -> Agent 处理分析 -> 另一个 Agent 校验 -> 存储组件（知识库或表格） -> 结束`

其中：

- 校验通过：进入存储并结束
- 校验不通过：回流到“Agent 处理分析”节点

这个例子要求工作流支持：

- 程序节点和 Agent 节点混编
- 条件回流
- 存储节点输出
- 节点状态更新和重试

## Agent 生命周期要求

工作流中的 Agent 需要支持至少三种生命周期：

1. **长期 Agent**
	- 工作区创建的长期 Agent
	- 拥有长期记忆

2. **工作流记忆 Agent**
	- 为某个工作流运行临时创建
	- 记忆在整个工作流执行期间保留
	- 工作流完成后记忆销毁或归档
	- 默认与 Workspace 长期记忆隔离

3. **一次性 Agent**
	- 只在单节点任务周期内存在
	- 节点完成后销毁
	- 记忆仅在当前节点有效

## 协作方向补充

- 工作流不应只支持线性流程，也应支持 `TaskMap / DAG` 风格的节点图。
- 节点之间除显式依赖外，还应支持事件钩子与资源绑定。
- 节点状态建议至少支持：`Pending / Ready / Claimed / Running / Blocked / Completed / Failed / Released / Skipped`。
- 编排者 Agent 可以维护任务图，但普通 Agent 也应能基于节点事件做局部探索与接管。
- 第一阶段先用 YAML 表达任务图与节点关系，后续再升级为图形化画板。

## 顺序任务

1. 定义最小 `WorkflowDefinition` / `WorkflowNode` / `WorkflowEdge` 模型
说明：建立 YAML / JSON 同构的工作流定义结构。
输出：最小工作流 schema。

2. 定义首批节点类型
说明：至少覆盖 Start、End、程序/脚本、Agent、Condition、存储、事件触发器。
输出：节点类型注册表。

3. 定义工作流触发模型
说明：支持 Agent、事件、用户、子工作流四类触发方式。
输出：`WorkflowTrigger` 模型与触发入口。

4. 定义工作流中的 Agent 生命周期模型
说明：支持长期 Agent、工作流记忆 Agent、一次性 Agent。
输出：节点级 Agent 生命周期配置。

5. 建立最小运行时执行链
说明：Controller 触发工作流，Runtime 执行节点，事件总线推进状态流转。
输出：首条可执行工作流链路。

6. 设计管理端工作流画布
说明：在 `PuddingPlatformAdmin` 中提供画布、节点表单、变量面板和运行记录页面。
输出：设计态界面方案。

## 验收标准

- 至少有 1 种 YAML 工作流定义格式。
- 至少支持 4 类基本节点：开始、结束、程序/脚本、Agent。
- 至少支持 1 个事件触发器节点。
- 至少支持 1 个工作流触发另一个工作流的链路。
- 至少支持 2 种 Agent 生命周期：长期 Agent、一次性 Agent。
- 管理端能表达工作流节点、边、节点配置和变量视图。
