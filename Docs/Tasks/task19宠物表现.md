# Task 19 — 宠物行为与多维交互设计

> **状态：** ✏️ 设计中
> **依赖：** Task 18 (产品定位)、Task 08 (记忆系统)、Task 09 (Agent 生命周期)
> **目标：** 定义布丁精灵的拟人化日常行为、记忆驱动的性格系统、多维感官交互方案及声音系统。
>
> **核心原则：** 模拟宠物日常不是增加用户负担（喂食/打扫），而是**赋予 AI 行为的合理性**。布丁在单机状态下也应极具生命力。

## 五、多维交互系统

### 5.1 物理触碰（直接交互）

让布丁具有实体感：

| 操作 | 布丁反应 |
| --- | --- |
| **文件投喂**（拖入文件） | 张嘴吞下，弹出分析卡片 |
| **链接捕获**（拖入 URL） | 变成潜水员潜入网页抓取信息 |
| **摸摸头**（单击/长按） | 咕噜声，身体泛起波纹 |
| **拎起来**（拖拽移动） | 四肢乱动，悬空挣扎形变 |
| **戳一戳**（双击） | 受惊弹开，物理碰撞效果 |

### 5.2 环形菜单（右键交互）

右键点击布丁，围绕它弹出一圈半透明图标气泡：

* **核心开关**：视觉感知、语音监听
* **状态切换**：专注模式、省电模式、待机休眠
* **记忆回溯**：查看近期习惯记录

### 5.3 语音对话（NLP 交互）

技术栈：Whisper.net（NPU 加速）+ Piper TTS

* **免唤醒**：特定场景下直接说「知道了」或「关掉它」，通过 VAD 识别
* **耳语模式**：深夜自动降低音量，使用气声
* **碎碎念**：长时间无交互时自言自语
* **语音视觉化**：说话时身体随声波频率波动

### 5.4 视觉注视（非接触交互）

技术栈：NPU + OpenVINO 注视点估算

* **眼神对视**：检测到注视时停止动作，转头对视，害羞变红
* **指点江山**：注视文件 + 语音指令 -> 布丁爬到文件旁询问操作
* **眨眼互动**：快速眨两下触发预设快捷功能

### 5.5 系统级感知（环境交互）

* **窗口碰撞**：被大窗口盖住时从边缘伸出半个头
* **通知指路**：后台程序有紧急通知时跑到对应任务栏图标上方敲门
* **剪贴板联动**：复制文字后手中变出发光小球，点击可翻译/改写/存入记忆

### 5.6 交互响应矩阵

| 触发 | 视觉反应 | 声音反馈 |
| --- | --- | --- |
| 检测到疲劳 | 爬到焦点窗口，打哈欠，拉横幅 | 柔和提示音 |
| 检测到站立 | 欢呼跳跃，桌面撒花 | 欢快音效 |
| 电量低进入省电 | 换睡衣，角落缩成球 | 呼吸灯变暗绿 |
| 用户长时间注视 | 身体变红，眼神闪躲，卖萌 | 轻微啾声 |

## 六、声音系统

### 6.1 TTS 语音

* **技术选型**：Piper TTS（ONNX，NPU 优化，即想即说）
* **音色**：幼态、软糯、带电子灵动感（小奶音风格 ONNX 模型）
* **情感语调**：兴奋时语速快音调上扬；提醒休息时柔和缓慢
* **语气词注入**：LLM 自动在句首加入语气助词增加真实感

### 6.2 交互音效 (SFX)

| 场景 | 音效 |
| --- | --- |
| 落地/碰撞 | 清脆 Q 弹噗啾声 |
| 拖拽形变 | 橡皮筋/粘液拉伸声 |
| 文件投喂 | 咕噜吞咽声 |
| 长按抚摸 | 低频呼噜声（Purring） |
| 习惯达成 | 叮水滴声 |

### 6.3 多模态同步

* **体振 Lip-sync**：无复杂嘴型，通过身体颤动幅度 + 光晕闪烁频率对应声音振幅（FFT）
* **空间音效**：布丁在屏幕左边从左声道发声；拖到右边声音随之漂移

## 七、技术架构

| 模块 | 技术栈 | 职责 |
| --- | --- | --- |
| **行为状态机** | C# State Machine | PuddingState（Sleep / Playful / Productive / Cautious） |
| **环境感知** | 系统 API + FlaUI | 时间/天气、CPU 负载、窗口坐标 |
| **记忆反馈** | MemoryHub + LLM | 分析 Profile.md 习惯标签，定时发送 BehaviorSignal |
| **视觉感知** | NPU + OpenVINO | 注视点追踪、表情分析、疲劳/喝水/站立检测 |
| **物理引擎** | C# 软体模拟 | 窗口碰撞、惯性掉落、软体形变 |
| **声音系统** | Piper TTS + NAudio | 情感化语音、碰撞音效、空间音频 |
| **能效管理** | PowerMode Listener | AC/DC 自动切换，电池模式下 NPU 极低功耗 |

```csharp
// 伪代码：情感语音播放
public async Task PuddingSpeakAsync(string text, EmotionType emotion)
{
    // 1. LLM 根据情感添加语气助词
    var emotionalText = AddToneMarkers(text, emotion);

    // 2. 调用 Piper 在 NPU 上生成语音流
    var audioStream = await PiperEngine.GenerateAudioAsync(emotionalText, "pudding_voice.onnx");

    // 3. 视觉联动：让布丁开始随着音频频率抖动
    VisualEngine.StartVoiceSync(audioStream);

    // 4. 播放
    AudioPlayer.Play(audioStream);
}
```
