# Task 20：物理引擎 — 物理感知型桌面生物

## 概述

实现一个"物理感知型桌面生物"：将 **Windows 窗口元数据** 转化为 **2D 物理世界的碰撞体**，让布丁精灵真正"住"在 Windows 桌面里，能在窗口上行走、攀爬、掉落，并拥有 Q 弹的软体质感。

---

## 一、桌面物理映射 (Desktop Physics Mapping)

在后台构建一个看不见的物理模拟层，让精灵能在窗口上"逛街、攀爬"。

### 1.1 窗口碰撞箱同步

利用 `Win32 API` 实时获取所有活跃窗口的 `Rect`（坐标与大小），在物理引擎（如 **Box2D** 或 **Chipmunk2D**）中为每个窗口生成对应的静态刚体。

### 1.2 重力与摩擦力

给布丁设定向下的全局重力。当它站在窗口顶部时，物理引擎通过"支撑力"让它停留在那里。

### 1.3 惯性与联动

当用户拖动窗口时，捕捉窗口的位移速度（Velocity）。如果窗口突然停下或剧烈晃动，物理引擎计算动量，布丁会因惯性产生滑行、翻滚甚至从窗口边缘"掉落"。

---

## 二、软体物理模拟 (Soft Body Physics)

使用 **网格变形（Mesh Deformation）** 实现"黏糊糊、Q 弹"的布丁质感。核心是将布丁看作由多个质点组成的弹性网格。

### 2.1 弹簧质点系统 (Spring-Mass System)

* **质点 (Mass Points)**：在布丁轮廓上均匀分布一组质点（16 个点组成圆圈），每个点有质量和速度。
* **弹簧 (Springs)**：
  * **外周弹簧**：维持布丁轮廓。
  * **内部支撑弹簧**：质点连接到中心点或互相交叉连接，防止塌陷。
* **阻尼 (Damping)**：加入较高的阻尼系数，让动能快速衰减，产生厚重的果冻感。

### 2.2 黏着力与形变 (Stickiness & Deformation)

* **表面吸附 (Surface Tension)**：质点靠近窗口边缘时施加微小吸引力，离开时产生"拉丝"形变。
* **压力保持 (Pressure Constraint)**：计算网格面积，被压缩时向外施压保持体积。
* **非线性形变**：用 **Bezier 曲线** 或 **Catmull-Rom 插值** 连接质点，8 个质点即可呈现圆润曲面。

### 2.3 碰撞形变表现

| 场景 | 表现 |
|------|------|
| 落地 | 底部顶点向上收缩，侧面向外扩张，产生"啪叽"果冻感 |
| 鼠标挤压 | 受力点凹陷，其余向外扩张（保持体积） |
| 攀爬拉伸 | 身体拉长，呈"水滴"形状 |
| 窗口晃动 | 质点滞后移动，整体"甩"变形，可能掉落 |

## 三、AI 自主行为系统

### 3.1 意图驱动 (Intent-Driven)

布丁的动作由 LLM 的"后台思绪"驱动，而非预设的随机循环。

* **思绪引擎 (Thought Stream)**：后台 LLM 定期"闲思"。例如扫描到 `MEMORY.md` 提到下午有会，意图从"闲逛"转为"准备提醒"。
* **好奇心模型**：对桌面上新出现的文件产生"好奇"，爬过去观察但 **绝不自动打开**，只用"歪头"动作吸引注意。

### 3.2 三层防御架构 (The Three Shields)

在 LLM 意图与物理执行之间加入过滤器，确保不干扰工作。

**第一层：视觉隔离 (Visual Non-Interference)**

* **避让算法**：利用 UI Automation 感知文本输入区域和鼠标活跃区，自主绕行避让。
* **半透明化回避**：全屏游戏或观影时自动进入"深度睡眠"并淡化到几近透明，甚至缩入系统托盘。

**第二层：物理交互约束 (Collision Etiquette)**

* **软碰撞**：对活动窗口的碰撞力极小，撞到窗口时自己形变弹开，不导致窗口抖动。
* **静音规则**：除紧急提醒外，所有自主行为无声。

**第三层：操作权限沙箱 (Operation Sandbox)**

* **禁区设置**：系统关键目录（System32 等）为逻辑禁区。
* **确认握手**：真正的操作（如删除文件）必须等待用户点击确认。

### 3.3 趣味性自主行为

* **寻光性**：偏好屏幕上亮度最高或颜色最暖的区域。
* **整理癖**：桌面图标太乱时走到旁边做"推"的动作，露出无奈表情。
* **记忆共鸣**：通过系统媒体 API 感知正在播放的歌曲，闭眼随节奏轻微摇摆。
* **窗口攀爬**：长时间活跃的窗口上，爬到标题栏坐着。
* **图标交互**：跳到文件夹图标上踩一踩，在回收站旁边做"嫌弃"的躲避动作。
* **受惊反馈**：快速晃动窗口时进入受惊状态（`>_<`），缩成一团球。

### 3.4 用户训导

* **"嘘——"操作**：点击嘴巴，1 小时内进入绝对静默态。
* **"去那边"**：拎起扔到屏幕另一角，它会记住该位置为当前"禁区"。

## 四、视觉特效

### 4.1 着色器特效 (Shaders)

* **折射与透光 (Refraction)**：半透明布丁让背景产生微小扭曲，增强"果冻感"。
* **高光动态 (Specular Highlight)**：形变时高光点随之移动，暗示光滑湿润表面。
* **拖尾特效 (Slime Trail)**：快速爬行时留下逐渐消失的淡透明路径。

### 4.2 碰撞与粒子效果

* **接触光晕**：碰撞窗口边缘时产生淡黄色微弱辉光。
* **布丁碎屑粒子**：掉落弹起时溅出半透明粒子，随后消失。
* **鼠标涟漪**：鼠标移到身上时产生水面涟漪扩散特效。

### 4.3 实时投影 (Drop Shadow)

投影根据布丁与下方物体的距离动态虚化，非固定阴影。

## 五、技术架构

### 5.1 双层状态机

| 层级 | 职责 | 运行方式 |
|------|------|----------|
| 底层 (Physics & Logic) | 重力、碰撞、避让逻辑 | C# 高性能独立线程 |
| 上层 (LLM Intelligence) | 定期或事件触发产生 `Intent`（如 `LOOK_AT_NEW_FILE`、`STRETCH`） | 底层判断环境是否允许执行 |

### 5.2 物理引擎选型

#### 方案 A：自研轻量级引擎（推荐）

使用 **Verlet 积分器**，计算量极小，易于自定义"黏度"。

```csharp
// Verlet 积分更新质点位置
void UpdatePoint(Point p)
{
    var temp = p.Position;
    p.Position += (p.Position - p.OldPosition) + p.Acceleration * dt * dt;
    p.OldPosition = temp;
}
```

#### 方案 B：集成第三方引擎 (Box2D.NET / Chipmunk)

碰撞检测成熟，用 `DistanceJoint`（距离关节）模拟弹簧，适合多窗口重叠的复杂场景。

### 5.3 物理循环伪代码

```csharp
public void UpdatePhysics()
{
    // 1. 获取当前焦点窗口及周边窗口信息
    var activeWindowRect = WindowsAPI.GetActiveWindowRect();

    // 2. 将 Windows 坐标转换为物理引擎坐标
    physicsWorld.UpdatePlatform(activeWindowRect);

    // 3. 计算布丁的软体形变
    puddingSoftBody.ApplyForce(gravity);

    // 4. 如果检测到窗口位移速度过快，触发掉落
    if (windowVelocity > threshold)
    {
        puddingSoftBody.State = Falling;
    }
}
```

## 六、性能与约束

* **线程隔离**：物理计算放在独立线程，平时安静时切回简单帧动画以节省电力。
* **穿透点击**：利用 `WS_EX_LAYERED` + `WS_EX_TRANSPARENT` 属性，确保桌面走动时不干扰点击下方图标，仅特定交互（如拖拽文件）时截获鼠标。

## 设计理念

> 布丁的自主性是"观察者"和"陪伴者"的自主，而不是"操作者"的自主。它像一只懂事的猫，会好奇地看着你工作，会在桌面上玩耍，但当你全神贯注时，它会安静地蜷缩在角落。
