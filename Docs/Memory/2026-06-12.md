# 2026-06-12 工作日志

## 研究发现
- 用户补充：PuddingAgent 不能假设只有一个类似 Codex/Claude 启动目录的项目根；一个 workspace 需要支持多个项目目录的添加、移除、列表、检索和索引。
- 用户补充：项目目录不一定在 workspace 目录树下。workspace 是注册、会话和授权上下文，不是源代码文件系统边界。
- 用户补充：rename 只是 IDE 的一个功能，不能把编辑服务设计成 rename-only。
- ADR-049 已调整为 `ICodeProjectRegistry` + `ICodeIdeActionService`：项目目录通过工具注册，查询默认覆盖 workspace 下已注册项目，IDE action apply 以 registered project root 为源码写入边界。
- 实施计划已调整：新增 `CodeProjects`、`code_project_add/remove/list`、`ProjectId` 字段、workspace-keyed 索引 DB；项目目录可以是任意本机目录，但必须通过工具注册和权限校验。
- 用户指定本轮执行方式：Inline Execution，但以串行子代理方式推进，直到任务和目标验收完成。
- 用户指定模型偏好：编码子代理优先 5.3-codex 和 5.3-codex-spark，轻量任务用 5.3-codex-spark。
- 用户收缩本轮目标：实现项目骨架、SQLite 代码图、LSP 服务集成、项目目录添加/移除、项目目录解析和索引、Roslyn C# 索引器、只读查询工具；rename 预览/执行、模板暴露、完整验证等高级功能暂不做。
- 已新增本轮专用计划：`Docs/superpowers/plans/2026-06-12-code-intelligence-core-mvp.md`。

## 待确认
- `code_project_add` 对任意本机目录的授权策略：是否所有 agent 都能注册，还是外部目录需要 runtime approval。
- `code_project_remove` 是否应默认暴露给内置模板；它不删源代码，但会删除注册和索引状态。
- 后续 IDE action 列表优先级：rename 之后是否优先做 references/definition、format、organize imports、code action、extract method。
