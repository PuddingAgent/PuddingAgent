# 2026-06-11 工作日志

## 研究发现
- codegraph README 定位为本地预索引代码知识图谱，面向 Claude Code/Codex/Cursor 等 agent 暴露符号关系、调用图和结构化代码上下文。
- codegraph 当前公开能力包括本地 `.codegraph/` 索引、MCP server、SQLite、文件监听自动同步、impact analysis、symbol/call graph/code snippets，以及 20+ 语言声明支持。
- PuddingAgent 的本轮目标比 codegraph 更强调语言服务级编辑工具，例如 rename class/variable 应通过 LSP/Roslyn 重构能力执行，而不是文本替换。
- 已克隆 `temp/codegraph-reference` 做源码阅读。codegraph 核心实现是 tree-sitter 抽取 + SQLite `nodes/edges/files/unresolved_refs/nodes_fts` + reference resolution + MCP tools。
- codegraph 的公开 MCP 工具偏查询：explore/search/callers/callees/impact/node/files/status；未发现 `textDocument/rename`、workspace edit、Roslyn rename 或 refactor 执行能力。
- C# 不能只用 tree-sitter 作为权威语义源。首版应使用 Roslyn `MSBuildWorkspace`/`SemanticModel`/`SymbolFinder`/`Renamer.RenameSymbolAsync`，tree-sitter 更适合后续多语言结构索引补充。
- PuddingAgent 现有工具层适合接入代码智能：`IPuddingTool` + `[Tool]` + `ToolSafetyFlags` + runtime approval。只读查询工具可以 read-only；rename/refactor 必须 `High` + `RequiresFileWrite | Destructive`。
- 建议新增 `Source/PuddingCodeIntelligence`，把索引、SQLite code graph、Roslyn/LSP 适配和编辑服务隔离出来，Runtime 只负责工具暴露和后台服务注册。
- 已新增 ADR 草案：`Docs/07架构/50ADR-049代码语义索引与LSP编辑服务ADR.md`。
- 已新增实现计划：`Docs/superpowers/plans/2026-06-11-code-intelligence-mvp.md`，覆盖项目骨架、SQLite store、Roslyn C# indexer、只读工具、prepare rename、rename apply、模板暴露和验证。
