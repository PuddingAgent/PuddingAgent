# 编译和打包过程优化施工方案

> 日期：2026-05-23  
> ADR：[41ADR-040开发构建与发布打包链路分离ADR](../07架构/41ADR-040开发构建与发布打包链路分离ADR.md)  
> 目标：把高频开发调试从生产构建和 Docker 镜像构建中拆出来。开发模式不依赖 Docker，由脚本在宿主机后台启动后端 `dotnet watch` 和前端 `pnpm run start:dev`；Docker 仅用于快速集成和发布验证。

---

## 1. 施工原则

1. **开发模式不依赖 Docker**：日常开发不执行 `docker compose up`、`docker compose build`、`dotnet publish`、`pnpm run build`。
2. **后台进程可控**：`dev-up.ps1` 必须支持启动、停止、查看日志、状态检查，PID 和日志统一放到 `tmp/dev/`。
3. **发布链路不降级**：现有 `build-and-up.ps1` 普通模式仍能完成生产构建和镜像启动。
4. **先新增后替换**：第一阶段只新增 `dev-up.ps1`，确认稳定后再优化现有 `build-and-up.ps1`。
5. **装配职责收敛**：Admin SPA 生产产物最终按 ADR-037 由 `PuddingAgent.csproj` 装配，不再由 Dockerfile 和脚本分别复制。

## 4. Task 1：新增宿主机开发启动脚本

**文件：**

- Create: `dev-up.ps1`

**行为要求：**

1. 启动前检查 `dotnet`、`node`、`corepack`、`pnpm` 是否可用；
2. 启动前检查端口 `5000` 和 `8000` 是否被占用；
3. 准备 `data/config` 默认配置；
4. 后台启动后端 `dotnet watch`；
5. 后台启动前端 `pnpm run start:dev`；
6. PID 写入 `tmp/dev/backend.pid` 和 `tmp/dev/frontend.pid`；
7. 日志写入 `tmp/dev/backend.out.log`、`tmp/dev/backend.err.log`、`tmp/dev/frontend.out.log`、`tmp/dev/frontend.err.log`；
8. 支持 `-Down` 停止后台进程；
9. 支持 `-Logs` 跟随日志；
10. 支持 `-Status` 输出进程状态；
11. 支持 `-Restart` 先停止再启动；
12. 支持 `-NoInstall` 跳过前端依赖安装。

**建议内容：**

```powershell
# ============================================================
# Pudding Agent - 本地开发 watch 启动脚本
# ============================================================
# 用法：
#   .\dev-up.ps1
#   .\dev-up.ps1 -Status
#   .\dev-up.ps1 -Logs
#   .\dev-up.ps1 -Down
#   .\dev-up.ps1 -Restart
#   .\dev-up.ps1 -NoInstall

param(
    [switch]$Down,
    [switch]$Logs,
    [switch]$Status,
    [switch]$Restart,
    [switch]$NoInstall
)

$ErrorActionPreference = "Stop"
$Root = $PSScriptRoot
$RunDir = Join-Path $Root "tmp\dev"
$BackendPidFile = Join-Path $RunDir "backend.pid"
$FrontendPidFile = Join-Path $RunDir "frontend.pid"
$BackendOutLog = Join-Path $RunDir "backend.out.log"
$BackendErrLog = Join-Path $RunDir "backend.err.log"
$FrontendOutLog = Join-Path $RunDir "frontend.out.log"
$FrontendErrLog = Join-Path $RunDir "frontend.err.log"

function Step($msg) { Write-Host "`n==> $msg" -ForegroundColor Cyan }
function Ok($msg) { Write-Host "    V $msg" -ForegroundColor Green }
function Warn($msg) { Write-Host "    ! $msg" -ForegroundColor Yellow }
function Fail($msg) { Write-Host "    X $msg" -ForegroundColor Red; exit 1 }

function Require-Command($name) {
    if (-not (Get-Command $name -ErrorAction SilentlyContinue)) {
        Fail "缺少命令：$name。请先安装并确保它在 PATH 中。"
    }
}

function Get-PidFromFile($path) {
    if (-not (Test-Path $path)) { return $null }
    $raw = (Get-Content $path -ErrorAction SilentlyContinue | Select-Object -First 1)
    $pidValue = 0
    if ([int]::TryParse($raw, [ref]$pidValue)) { return $pidValue }
    return $null
}

function Test-ProcessAlive($pidValue) {
    if (-not $pidValue) { return $false }
    return [bool](Get-Process -Id $pidValue -ErrorAction SilentlyContinue)
}

function Stop-ProcessTree($pidValue) {
    $children = Get-CimInstance Win32_Process -Filter "ParentProcessId = $pidValue" -ErrorAction SilentlyContinue
    foreach ($child in $children) {
        Stop-ProcessTree $child.ProcessId
    }
    $process = Get-Process -Id $pidValue -ErrorAction SilentlyContinue
    if ($process) {
        Stop-Process -Id $pidValue -Force
    }
}

function Stop-TrackedProcess($name, $pidFile) {
    $pidValue = Get-PidFromFile $pidFile
    if (-not $pidValue) {
        Warn "$name 未记录 PID"
        return
    }
    $process = Get-Process -Id $pidValue -ErrorAction SilentlyContinue
    if ($process) {
        Stop-ProcessTree $pidValue
        Ok "已停止 $name (PID $pidValue)"
    } else {
        Warn "$name PID $pidValue 已不存在"
    }
    Remove-Item $pidFile -Force -ErrorAction SilentlyContinue
}

function Test-PortFree($port) {
    $listeners = Get-NetTCPConnection -LocalPort $port -State Listen -ErrorAction SilentlyContinue
    return -not $listeners
}

function Show-Status {
    $backendPid = Get-PidFromFile $BackendPidFile
    $frontendPid = Get-PidFromFile $FrontendPidFile
    $backendState = if (Test-ProcessAlive $backendPid) { "running (PID $backendPid)" } else { "stopped" }
    $frontendState = if (Test-ProcessAlive $frontendPid) { "running (PID $frontendPid)" } else { "stopped" }
    Write-Host "Backend : $backendState"
    Write-Host "Frontend: $frontendState"
}

function Prepare-Config {
    $configDir = Join-Path $Root "data\config"
    $defaultConfigDir = Join-Path $Root "Source\PuddingAgent\default-data\config"
    if (-not (Test-Path $configDir)) {
        New-Item -ItemType Directory -Path $configDir -Force | Out-Null
    }
    foreach ($name in @("llm.providers.json", "system.json", "security.json", "connectors.json")) {
        $target = Join-Path $configDir $name
        $source = Join-Path $defaultConfigDir $name
        if ((-not (Test-Path $target)) -and (Test-Path $source)) {
            Copy-Item -Path $source -Destination $target
            Write-Host "[i] 已创建默认配置 data/config/$name" -ForegroundColor Yellow
        }
    }
}

function Start-Backend {
    Step "启动后端 dotnet watch"
    $env:ASPNETCORE_ENVIRONMENT = "Development"
    $env:ASPNETCORE_URLS = "http://localhost:5000"
    $env:DOTNET_USE_POLLING_FILE_WATCHER = "1"
    $env:PUDDING_DATA_ROOT = Join-Path $Root "data"

    $backendArgs = @(
        "watch",
        "--project",
        "Source\PuddingAgent\PuddingAgent.csproj",
        "run",
        "--urls",
        "http://localhost:5000"
    )

    $process = Start-Process `
        -FilePath "dotnet" `
        -ArgumentList $backendArgs `
        -WorkingDirectory $Root `
        -RedirectStandardOutput $BackendOutLog `
        -RedirectStandardError $BackendErrLog `
        -WindowStyle Hidden `
        -PassThru

    Set-Content -Path $BackendPidFile -Value $process.Id -Encoding ASCII
    Ok "后端已启动 (PID $($process.Id))"
}

function Start-Frontend {
    Step "启动前端 dev server"
    $adminDir = Join-Path $Root "Source\PuddingPlatformAdmin"
    $installCommand = if ($NoInstall) { "" } else { "corepack enable; pnpm install --frozen-lockfile; if (`$LASTEXITCODE -ne 0) { pnpm install; }" }
    $command = "$installCommand `$env:REACT_APP_ENV='dev'; `$env:MOCK='none'; `$env:UMI_ENV='dev'; `$env:PUDDING_API_BASE_URL='http://localhost:5000'; pnpm run start:dev -- --host 127.0.0.1 --port 8000"

    $process = Start-Process `
        -FilePath "powershell" `
        -ArgumentList @("-NoProfile", "-ExecutionPolicy", "Bypass", "-Command", $command) `
        -WorkingDirectory $adminDir `
        -RedirectStandardOutput $FrontendOutLog `
        -RedirectStandardError $FrontendErrLog `
        -WindowStyle Hidden `
        -PassThru

    Set-Content -Path $FrontendPidFile -Value $process.Id -Encoding ASCII
    Ok "前端已启动 (PID $($process.Id))"
}

New-Item -ItemType Directory -Path $RunDir -Force | Out-Null

if ($Down -or $Restart) {
    Step "停止开发进程"
    Stop-TrackedProcess "Frontend" $FrontendPidFile
    Stop-TrackedProcess "Backend" $BackendPidFile
    if ($Down -and (-not $Restart)) { exit 0 }
}

if ($Status) {
    Show-Status
    exit 0
}

if ($Logs) {
    Get-Content $BackendOutLog, $BackendErrLog, $FrontendOutLog, $FrontendErrLog -Wait -ErrorAction SilentlyContinue
    exit 0
}

Require-Command "dotnet"
Require-Command "node"
Require-Command "corepack"
Require-Command "pnpm"

if (-not (Test-PortFree 5000)) { Fail "端口 5000 已被占用，请先停止占用进程或修改开发端口。" }
if (-not (Test-PortFree 8000)) { Fail "端口 8000 已被占用，请先停止占用进程或修改开发端口。" }

Prepare-Config
Start-Backend
Start-Frontend

Write-Host @"

开发环境已启动：
  后端 API  -> http://localhost:5000
  前端 Dev -> http://localhost:8000

查看状态：
  .\dev-up.ps1 -Status

查看日志：
  .\dev-up.ps1 -Logs

停止：
  .\dev-up.ps1 -Down
"@ -ForegroundColor Yellow
```

**验证：**

```powershell
.\dev-up.ps1
.\dev-up.ps1 -Status
```

预期：

1. 输出后端和前端均为 `running`；
2. `tmp/dev/backend.pid` 和 `tmp/dev/frontend.pid` 存在；
3. `tmp/dev/*.log` 存在；
4. `http://localhost:5000` 可访问后端；
5. `http://localhost:8000` 可访问前端 dev server。

## 7. Task 4：优化 `build-and-up.ps1 -Fast`

**文件：**

- Modify: `build-and-up.ps1`

**新增参数：**

```powershell
param(
    [switch]$Fast,
    [switch]$BuildOnly,
    [switch]$Restart,
    [switch]$NoCache,
    [switch]$SkipFrontend,
    [switch]$SkipBackend,
    [switch]$NoInstall
)
```

**前端构建规则：**

```powershell
if ((-not $Restart) -and (-not $SkipFrontend)) {
    Step "编译 Admin 前端 (pnpm)"
    $adminDir = "$Root\Source\PuddingPlatformAdmin"
    Invoke-Native {
        Push-Location $adminDir
        try {
            if (-not $NoInstall) {
                pnpm install --frozen-lockfile 2>$null
                if ($LASTEXITCODE -ne 0) {
                    Write-Host "    pnpm install --frozen-lockfile 失败，尝试 pnpm install..." -ForegroundColor Yellow
                    pnpm install
                }
            }
            pnpm run build
        }
        finally {
            Pop-Location
        }
    } "Admin 前端编译失败"
    Ok "Admin 前端编译通过"
}
```

**后端 publish 规则：**

```powershell
if ($Fast -and (-not $Restart) -and (-not $SkipBackend)) {
    Step "编译 PuddingAgent（发布模式，用于卷挂载）"
    $publishDir = "$Root\Source\PuddingAgent\bin\Release\net10.0\publish"
    if (Test-Path $publishDir) {
        Remove-Item "$publishDir\*" -Recurse -Force -ErrorAction SilentlyContinue
    }
    Invoke-Native {
        dotnet publish "$Root\Source\PuddingAgent\PuddingAgent.csproj" -c Release -o $publishDir --nologo
    } "编译失败"
    Ok "编译通过（发布产物: $publishDir）"
}
```

**注意：**

这一任务只优化快速集成链路。日常开发已经由 `dev-up.ps1` 承担，不应让 `build-and-up.ps1 -Fast` 再承担高频调试职责。

**验证：**

```powershell
.\build-and-up.ps1 -Fast -SkipFrontend
.\build-and-up.ps1 -Fast -SkipBackend
.\build-and-up.ps1 -Fast -NoInstall
.\build-and-up.ps1 -Fast -Restart
```

预期：

1. `-SkipFrontend` 不输出“编译 Admin 前端”；
2. `-SkipBackend` 不执行 `dotnet publish`；
3. `-NoInstall` 不执行 `pnpm install`；
4. `-Restart` 仍只重启容器。

## 9. Task 6：减少发布链路重复编译

**文件：**

- Modify: `build-and-up.ps1`
- Modify: `Source/PuddingAgent/Dockerfile`

**建议策略：**

普通发布验证模式不要先宿主机 `dotnet build` 再 Docker 内 `dotnet publish`。二选一：

### 方案 A：Docker 是发布构建唯一入口

`build-and-up.ps1` 普通模式：

1. 执行前端 `pnpm run build`；
2. 不执行宿主机 `dotnet build`；
3. 执行 `docker compose build pudding-agent`；
4. 执行 `docker compose up -d`。

适合本地模拟发布镜像。

### 方案 B：增加 `-Validate` 参数

只有显式传入 `-Validate` 时才执行宿主机：

```powershell
dotnet build "$Root\Source\PuddingAgent\PuddingAgent.csproj" -c Release --nologo
```

默认普通模式不重复 build。

**推荐**：采用方案 B，保留需要时本地验证能力，但默认消除重复编译。

**验证：**

```powershell
.\build-and-up.ps1
.\build-and-up.ps1 -Validate
```

预期：

1. 不带 `-Validate` 时，宿主机不执行 `dotnet build`；
2. 带 `-Validate` 时，先执行宿主机 `dotnet build` 再构建镜像；
3. 两种模式均能启动服务。

## 10. Task 7：Dockerfile 缓存优化

**文件：**

- Modify: `Source/PuddingAgent/Dockerfile`

**建议内容：**

使用 BuildKit cache mount 缓存 NuGet 包：

```dockerfile
RUN --mount=type=cache,target=/root/.nuget/packages \
    dotnet restore Source/PuddingAgent/PuddingAgent.csproj

RUN --mount=type=cache,target=/root/.nuget/packages \
    dotnet publish Source/PuddingAgent/PuddingAgent.csproj \
    -c Release -o /app \
    --no-restore \
    --nologo
```

保留先复制 `.csproj` 再 restore，再复制源码的分层结构。

**注意：**

当前 Dockerfile 中 `ARG CACHE_BUST` 注释写着强制每次重新复制/重编，但 compose 未传入该参数时不会主动破坏缓存。施工时应删除误导性注释，不需要引入 cache bust。

**验证：**

```powershell
docker builder prune --filter type=exec.cachemount --force
docker compose build pudding-agent
docker compose build pudding-agent
```

预期：第二次构建 restore 层明显更快，且不重新下载全部 NuGet 包。
