# Pudding UI/UX 风格与交互改造

> 日期：2026-05-03  
> 范围：登录页、独立 Chat 页、后台控制台入口

## 设计定位

Pudding 的主界面不是后台系统，而是本地运行的私人 AI 代理。整体风格定义为“静谧书斋式 AI Agent”：安静、克制、专业、私有、有温度。

## UX 第一性原理

用户打开 Pudding 的首要目标是把意图交给 Agent 并获得可靠结果，而不是管理系统。因此：

- Chat 是主舞台，后台是低频工具箱。
- 登录后默认进入独立 Chat 页。
- 后台入口应收敛为 Chat 页中的“控制台”入口。
- 工作区和 Agent 选择应作为对话上下文，而不是后台筛选器。

## 当前落地

- `/chat` 使用独立布局，不再继承 Ant Design Pro 的侧栏、顶栏、Footer 与水印。
- Chat 页增加 Pudding 品牌页头、当前上下文胶囊和独立上下文选择条。
- 登录页品牌从 `Pudding Platform` 调整为 `Pudding`。
- 后台示例页清理 Ant Design Pro 模板内容，定位为 `Pudding Console`。
- 全局按钮 hover 动效改为克制的颜色/阴影过渡，移除放大动效。

## 后续阶段

1. 将工作区/Agent 双选择器收敛为一个上下文浮层。
2. 增加会话历史侧栏或抽屉。
3. 定义完整 Pudding Design Tokens，统一登录页、Chat 页与后台控制台。
4. 后台菜单与表单/表格继续降噪，使其更像设置抽屉而不是企业 CRM。
