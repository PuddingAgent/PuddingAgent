# 启动和调试

dev-up.ps1

第一次冷启动，需要先完成初始化，请跳转到/admin/bootstrap，完成初始化引导流程，之后就可以正常使用了。

如果你需要手工编译或者调试，请先使用dev-up.ps1停止守护，避免守护进程干扰你的调试过程。完成调试后，可以重新启动守护进程。

程序日志位于data\logs目录下，可以通过查看日志文件来获取程序的运行状态和调试信息。

如果你遇到一些麻烦的问题，不如多插入一些日志log诊断和跟踪过程，看看程序的运行状态，或者直接查看日志文件，看看有没有什么异常信息，这些都可以帮助你更好地理解程序的运行情况和调试过程。



# data目录

data目录包含了测试数据和配置文件，确保在启动服务时正确加载这些数据，以便进行测试和调试。第一次冷启动，此目录（data）需要是空的，软件本身应该会初始化必要的文件和目录结构。


如果需要清理环境（比如需要重新使用干净的目录），可以删除data目录下的内容，重新启动服务，软件会重新初始化必要的文件和目录结构。
请确保在启动服务之前，data目录具有适当的权限，以便软件能够正确读取和写入数据。
